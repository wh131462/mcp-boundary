# Boundary MCP

> 需求文档（PRD v0.1）

---

## 1. 项目背景与目标

### 1.1 背景

在现实决策中，用户面临的问题往往具备以下特征：

- 信息不完整
- 情绪参与度高
- 变量多、约束强
- 容易被叙事、样本偏差误导

传统 AI 或建议型工具通常：

- 给出单一结论
- 强行"替用户做决定"
- 忽略不确定性和结构性约束

### 1.2 项目目标

构建一个 **思考型 / 策略型 MCP**，用于：

- 将模糊、情绪化的现实问题
- 拆解为结构化、可推理的决策模型
- 帮助用户看清结构，而非替用户做选择

---

## 2. 产品定位

### 2.1 核心定位

> 一个"暴露现实结构"的决策思考引擎，而不是建议生成器

### 2.2 不做的事情（明确边界）

- ❌ 不直接告诉用户"该不该做"
- ❌ 不预测具体结果
- ❌ 不提供情绪安慰或鸡汤
- ❌ 不迎合用户已有立场

---

## 3. 设计原则（产品灵魂）

| 原则 | 说明 |
|------|------|
| 不做决定 | 只暴露结构 |
| 不给答案 | 只给边界 |
| 不预测未来 | 只给分布 |
| 不迎合情绪 | 只校正幻觉 |

> ⚠️ 这些原则在任何功能设计中都不可被破坏。

---

## 4. 用户画像

### 4.1 目标用户

- 面临重要人生 / 职业 / 策略决策的人
- 对"思考质量"要求高于"情绪安慰"
- 能接受复杂性与不确定性

### 4.2 使用场景示例

- 是否移民 / 换国家工作
- 是否转行 / All in 一个方向
- 是否进入某个高风险赛道
- 如何理解某个社会或政策变化

---

## 5. 核心功能需求

### 5.1 核心功能总览（MVP）

系统提供 **单一核心能力**：

> 对现实问题进行结构化策略分析

### 5.2 功能 1：问题去情绪化重述

**输入**

用户自然语言描述的问题（可能夹杂情绪、立场、焦虑）

**输出**

去除情绪、口号、预设结论后的问题重述

**示例输出**

```
重述后的核心问题：
在给定年龄、行业背景与政策环境下，选择海外工作的长期收益与风险结构是什么？
```

### 5.3 功能 2：关键变量与约束识别

**输出内容**

- 目标（用户真正关心的指标）
- 关键变量（可变化因素）
- 硬约束（不可忽视的现实限制）
- 不确定性来源

**示例结构**

```yaml
目标:
  - 长期收入稳定性
  - 身份与安全边际
  - 技术成长空间

关键变量:
  - 年龄
  - 行业周期
  - 技术可迁移性

硬约束:
  - 签证政策
  - 成本结构
  - 时间窗口

不确定性:
  - 政策变化
  - 全球经济周期
```

### 5.4 功能 3：策略空间生成

**目标**

- 避免"只有一个选项"的假问题
- 显式给出多个可选策略路径

**输出要求**

- 至少 3–4 种策略
- 每种策略包含：
  - 前提条件
  - 执行成本
  - 触发点
  - 最坏情况

**示例**

| 策略 | 类型 |
|------|------|
| 策略 A | 立即行动（激进） |
| 策略 B | 延迟行动（观望） |
| 策略 C | 双轨并行（风险对冲） |
| 策略 D | 放弃该路径（止损） |

### 5.5 功能 4：风险与回报分布分析

**设计重点**

- 不输出"成功 / 失败"
- 只输出概率分布与区间结果

**示例**

```
策略 C 的结果分布：
├── 30%：成本上升但机会保留
├── 40%：条件改善，收益中等
└── 30%：时间成本被浪费
```

### 5.6 功能 5：认知偏差与幻觉提示

**目的**

主动指出用户可能被哪些非理性因素影响

**常见类型**

- 幸存者偏差
- 叙事驱动决策
- 群体压力
- 对短期结果的过度关注

**示例输出**

```
⚠️ 潜在认知偏差：
- 将个别成功案例误认为普遍路径
- 低估制度摩擦成本
```

---

## 6. 技术实现范围（MVP）

### 6.1 MCP 工具设计

提供单一 Tool：

```typescript
analyze_decision(problem_text: string)
```

### 6.2 输出结构（固定）

1. 问题重述
2. 变量与约束
3. 策略集合
4. 风险 / 回报分布
5. 认知偏差提示

---

## 7. 非目标（明确不做）

- ❌ 不构建用户系统
- ❌ 不存储历史决策
- ❌ 不进行模型训练
- ❌ 不提供自动化结论推荐

---

## 8. 成功标准

该 MCP 是否成功，**不取决于**：

> 用户是否采纳某个策略

**而取决于**：

> 用户是否更清楚自己在赌什么

是否能说出：

> *"我现在知道我是在什么条件下做这个决定的"*
